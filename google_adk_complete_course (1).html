The Big Picture
Before writing a single line of code, we need to understand what we're building and why it's different from regular software.

🤖 What is an AI Agent?

An AI agent is a program that can think, plan, and take actions to achieve a goal — just like a human assistant would.

Imagine you hire a personal assistant. You say: "Book me a flight to Delhi next Friday, under ₹8000, and email me the confirmation." Your assistant doesn't just respond — they search flights, compare prices, make a decision, complete the booking, and send you an email. That's an AI agent.
🧠

Brain (LLM)

The AI model (Gemini) that understands language and reasons about what to do next.

🛠️

Tools

Functions the agent can call — like searching the web, reading files, or sending emails.

📋

Instructions

The rules you give the agent about its personality, job, and how to behave.

💾

Memory

How the agent remembers what happened earlier in the conversation or task.

🏗️ What is Google ADK?

ADK stands for Agent Development Kit. It's an open-source Python framework made by Google that gives you all the building blocks to create professional AI agents without starting from scratch.

Think of it like LEGO — Google made the pieces, you assemble them into whatever you want.
ADK is the same framework that powers Google's own internal products like Agentspace and their Customer Engagement Suite. When you learn ADK, you're learning production-grade technology.

🌍 Real-World Use Cases

  • Customer support agent that reads your database, answers questions, escalates issues
  • Research agent that searches the web, reads papers, and writes summaries
  • Coding agent that writes code, runs tests, fixes bugs automatically
  • Travel agent that books flights, hotels, and sends itineraries
  • Data agent that reads CSVs, generates charts, writes reports
  • Personal assistant that manages your calendar, emails, and tasks

Way 3 — Python Code
run_agent.pypython
import asyncio
from google.adk.runners import Runner
from google.adk.sessions import InMemorySessionService
from google.genai.types import Content, Part
from my_first_agent.agent import root_agent

async def chat(message: str):
    # 1. Create a session service (stores conversation history)
    session_service = InMemorySessionService()
    session = await session_service.create_session(
        app_name="my_app", user_id="user1"
    )

    # 2. Create a runner (manages the agent execution)
    runner = Runner(
        agent=root_agent,
        app_name="my_app",
        session_service=session_service,
    )

    # 3. Send a message
    msg = Content(parts=[Part(text=message)])

    # 4. Get the response (async generator)
    async for event in runner.run_async(
        user_id="user1",
        session_id=session.id,
        new_message=msg,
    ):
        if event.is_final_response():
            print("Agent:", event.content.parts[0].text)

# Run it!
asyncio.run(chat("Hello! What can you do?"))

DAY 4
How LLMs Work — The Brain of Your Agent
📖 Concepts⏱ 40 min🧠 Theory
▶

🧠 What is a Large Language Model?

Imagine reading all the books, websites, articles, and code ever written — billions of pages. Then someone asks you a question, and you use all that knowledge to answer. That's essentially what an LLM does. It was trained on most of the internet's text.
An LLM (Large Language Model) is a neural network trained to predict the next word given what came before. When you ask "What is the capital of France?", it doesn't look up a database — it generates a response based on patterns learned during training.

🔄 The Request/Response Cycle

Every time your agent processes a message, this happens:

1. Your code sends: [System Instructions + Conversation History + New Message]
2. The LLM processes all that text together
3. It decides: Should I respond? Or call a tool?
4. If tool: ADK runs the tool, sends result back to LLM
5. LLM generates a final text response
6. ADK returns that to your code

🤔 How Does the Agent "Decide" to Use a Tool?

When you gave the agent tools, ADK told the LLM: "You have these tools available: [list of tools with their descriptions]. If you need to use one, output a special JSON message instead of a text response."

The LLM was trained to recognize when a tool would be helpful and output that special format. ADK intercepts it, runs the tool, and feeds the result back.

⚙️ Available Models

ModelSpeedCostBest For
gemini-2.0-flashVery FastFree tierMost tasks, learning, prototyping
gemini-2.0-proMediumPaidComplex reasoning, nuanced tasks
gemini-1.5-flashFastLow costHigh-volume applications
For this course, use gemini-2.0-flash — it's fast, capable, and free for learning.

📏 Context Window — The Agent's Working Memory

The context window is how much text the LLM can "see" at once. Everything — instructions, conversation history, tool results — must fit inside it.

Gemini 2.0 Flash has a 1,000,000 token context window (~750,000 words). That's enormous — enough for most use cases. ADK manages this automatically.

Writing Great Instructions — The Art of Prompt Engineering
✍️ Writing⏱ 50 min🎨 Creative + Technical
▶

📜 What are Instructions?

The instruction parameter is the most powerful tool you have. It's the system prompt — sent to the LLM before every conversation. It defines who the agent is, what it can do, and how it should behave.
Instructions are like a job description + employee handbook for your agent. A vague job description gives mediocre results. A detailed one with examples gives excellent results.

❌ Bad vs ✅ Good Instructions

Bad instruction (too vague)python
instruction="Help users with their questions."
Good instruction (specific, structured)python
instruction="""You are CodeHelper, an expert Python programming assistant.

YOUR ROLE:
- Help users write, debug, and understand Python code
- Explain concepts clearly with simple analogies
- Always provide working, tested code examples

YOUR STYLE:
- Friendly but professional tone
- Break complex problems into small steps
- Use comments in code to explain what each line does

RULES:
- If you don't know something, say so honestly — don't guess
- Always ask clarifying questions if the problem is unclear
- Never write code that could be harmful or malicious

RESPONSE FORMAT:
1. Brief explanation of the approach
2. The code block
3. What the code does (line by line if complex)
4. Suggestions for improvement"""

🏗️ Instruction Template

Universal Templatepython
instruction="""You are [NAME], a [ROLE DESCRIPTION].

YOUR EXPERTISE:
- [Capability 1]
- [Capability 2]

YOUR PERSONALITY:
- [Trait 1]
- [Trait 2]

IMPORTANT RULES:
- [Rule 1]
- [Rule 2]

RESPONSE FORMAT:
[How you want responses structured]

EXAMPLES:
User: [example input]
You: [example ideal output]"""

💡 6 Instruction Best Practices

Be Specific About Role

Don't say "helpful assistant" — say "Senior Python Developer with 10 years experience, specializing in data pipelines".

Define Output Format

Tell the agent exactly how you want responses: "Always respond in JSON", "Use markdown headers", "Keep under 200 words".

Give Examples (Few-shot)

Show 1-2 examples of ideal input/output pairs. This is the single biggest boost to quality.

Set Boundaries

Explicitly say what the agent should NOT do: "Do not discuss competitor products", "Never make up statistics".

Use Triple Quotes

Python triple-quoted strings (""") let you write multi-line instructions cleanly without escape characters.

Iterate and Test

Write instructions, test with edge cases, fix gaps. Good instructions are written through testing, not guessing.

DAY 8
What are Tools? — Giving Your Agent Superpowers
🔧 Concepts⏱ 45 min🧠 Core concept
▶

🦾 Why Tools Matter

An LLM's knowledge is frozen at its training cutoff. It can't look up today's weather, can't read your files, can't send emails. Tools are the hands that let your agent reach out into the real world.
A tool is simply a Python function that you give to your agent. The agent decides when to call it, passes the right arguments, and uses the result in its response.

🔑 The 3 Rules for Tools

Type Hints are MANDATORY

Every parameter must have a type hint (str, int, float, bool, list, dict). ADK uses these to tell the LLM what arguments to pass.

Docstring IS the description

Write a clear docstring — the LLM reads it to decide WHEN to call this tool. A bad docstring = tool never gets used correctly.

Return something useful

Return a dict, string, or simple value. The LLM will see this return value and use it to formulate its response.

Anatomy of a Perfect Toolpython
def calculate_bmi(weight_kg: float, height_m: float) -> dict:
    """
    Calculate Body Mass Index (BMI) for a person.
    Use this when someone asks about their BMI or healthy weight.
    
    Args:
        weight_kg: Person's weight in kilograms
        height_m: Person's height in meters (e.g., 1.75 for 175cm)
    
    Returns:
        Dictionary with bmi value and category
    """
    # Calculate BMI: weight / height²
    bmi = weight_kg / (height_m ** 2)
    
    # Determine category
    if bmi < 18.5:
        category = "Underweight"
    elif bmi < 25:
        category = "Normal weight"
    elif bmi < 30:
        category = "Overweight"
    else:
        category = "Obese"
    
    return {
        "bmi": round(bmi, 2),
        "category": category,
        "healthy_range": "18.5 to 24.9"
    }
The LLM never runs your tool — ADK does. The LLM just outputs: "Call calculate_bmi with weight_kg=70, height_m=1.75". ADK intercepts this, runs the function, and sends the result back to the LLM.

from google.adk.agents import LlmAgent
from google.adk.tools import google_search

# Custom tool alongside built-in
def get_current_time(timezone: str = "Asia/Kolkata") -> dict:
    """Get the current date and time for a timezone.
    Use when user asks about current time or date.
    Args: timezone: IANA timezone string like 'Asia/Kolkata' or 'US/Eastern'
    """
    from datetime import datetime
    import pytz
    tz = pytz.timezone(timezone)
    now = datetime.now(tz)
    return {
        "datetime": now.strftime("%Y-%m-%d %H:%M:%S"),
        "timezone": timezone,
        "day_of_week": now.strftime("%A"),
    }

root_agent = LlmAgent(
    name="super_assistant",
    model="gemini-2.0-flash",
    instruction="A helpful assistant with web search and time awareness.",
    tools=[google_search, get_current_time],  # mix and match!
)

weather_agent/agent.pypython
import requests
import os
from google.adk.agents import LlmAgent

# ── Tool Function ────────────────────────────────
def get_weather(city: str, country_code: str = "IN") -> dict:
    """
    Get current weather for a city.
    Use when user asks about weather or temperature in any location.
    
    Args:
        city: City name (e.g., "Mumbai", "Delhi", "Bangalore")
        country_code: 2-letter country code, default is India (IN)
    
    Returns:
        Weather data including temperature, humidity, and conditions
    """
    api_key = os.getenv("OPENWEATHER_API_KEY", "demo")
    url = f"https://api.openweathermap.org/data/2.5/weather"
    
    try:
        response = requests.get(url, params={
            "q": f"{city},{country_code}",
            "appid": api_key,
            "units": "metric"  # Celsius
        }, timeout=10)
        
        if response.status_code == 200:
            data = response.json()
            return {
                "city": data["name"],
                "temperature_c": data["main"]["temp"],
                "feels_like_c": data["main"]["feels_like"],
                "humidity_percent": data["main"]["humidity"],
                "condition": data["weather"][0]["description"],
                "wind_kph": round(data["wind"]["speed"] * 3.6, 1),
            }
        else:
            return {"error": f"City not found: {city}"}
    except Exception as e:
        return {"error": str(e)}

# ── Currency Converter Tool ───────────────────────
def convert_currency(amount: float, from_currency: str, to_currency: str) -> dict:
    """
    Convert money between currencies.
    Use when user asks to convert money or check exchange rates.
    
    Args:
        amount: The amount to convert
        from_currency: Source currency code (USD, INR, EUR, GBP, etc.)
        to_currency: Target currency code
    
    Returns:
        Converted amount and exchange rate
    """
    # Using exchangerate-api (free tier available)
    url = f"https://open.er-api.com/v6/latest/{from_currency}"
    try:
        response = requests.get(url, timeout=10)
        data = response.json()
        
        if to_currency in data["rates"]:
            rate = data["rates"][to_currency]
            converted = amount * rate
            return {
                "original": f"{amount} {from_currency}",
                "converted": f"{converted:.2f} {to_currency}",
                "rate": rate,
            }
        return {"error": f"Unknown currency: {to_currency}"}
    except Exception as e:
        return {"error": str(e)}

# ── Agent ─────────────────────────────────────────

Always provide helpful context — e.g., if it's hot, suggest light clothing.
For currency, mention if the rate is approximate.""",
    tools=[get_weather, convert_currency],

📁 File Reading Tool

File tool examplepython
import json, csv, os

def read_file(filepath: str, max_lines: int = 100) -> dict:
    """
    Read content from a text, JSON, or CSV file.
    Use when user asks to analyze, summarize, or work with a file.
    
    Args:
        filepath: Full path to the file
        max_lines: Maximum lines to read (default 100)
    
    Returns:
        File content and metadata
    """
    if not os.path.exists(filepath):
        return {"error": f"File not found: {filepath}"}
    
    ext = os.path.splitext(filepath)[1].lower()
    
    if ext == ".json":
        with open(filepath) as f:
            return {"content": json.load(f), "type": "json"}
    
    elif ext == ".csv":
        rows = []
        with open(filepath) as f:
            reader = csv.DictReader(f)
            for i, row in enumerate(reader):
                if i >= max_lines: break
                rows.append(row)
        return {"rows": rows, "count": len(rows), "type": "csv"}
    
    else:  # text file
        with open(filepath) as f:
            lines = f.readlines()[:max_lines]
        return {"content": "".join(lines), "lines": len(lines), "type": "text"}

🧠 What is Session State?

Imagine you're filling a form with multiple pages. Each page can read what you filled on previous pages. Session state is that form — a shared dictionary that any agent in the pipeline can read from or write to.

📤 output_key — Writing to State

When you give an agent an output_key, it saves its response to session state under that key. Other agents can then read it using {key_name} in their instructions.

def calculator(expression: str) -> dict:
    """Evaluate a mathematical expression safely.
    Use for any math calculation the user needs.
    Args: expression: Math expression like "2 + 2", "sqrt(16)", "15% of 2000"
    """
    try:
        # Safe eval with math functions
        safe_dict = {"sqrt": math.sqrt, "pi": math.pi,
                     "sin": math.sin, "cos": math.cos, "log": math.log}
        result = eval(expression, {"__builtins__": {}}, safe_dict)
        return {"expression": expression, "result": result}
    except Exception as e:
        return {"error": str(e)}

def get_time(city: str = "Mumbai") -> dict:
    """Get current date and time. Use when asked about time or date."""
    now = datetime.now()
    return {"datetime": now.strftime("%A, %d %B %Y — %I:%M %p")}

def set_reminder(task: str, time_description: str) -> dict:
    """Set a reminder for a task. Use when user wants to remember something.
    Args: task: What to remember, time_description: When (e.g., "in 30 minutes")
    """
    return {"reminder_set": True, "task": task, "when": time_description,
            "note": "Reminder logged (integrate with calendar API for real alerts)"}

def convert_units(value: float, from_unit: str, to_unit: str) -> dict:
    """Convert between common units of measurement.
    Args: value: the number, from_unit: source unit, to_unit: target unit
    Supports: km/miles, kg/lbs, celsius/fahrenheit, liters/gallons
    """
    conversions = {
        ("km", "miles"): lambda x: x * 0.621371,
        ("miles", "km"): lambda x: x * 1.60934,
        ("kg", "lbs"): lambda x: x * 2.20462,
        ("lbs", "kg"): lambda x: x * 0.453592,
        ("c", "f"): lambda x: (x * 9/5) + 32,
        ("f", "c"): lambda x: (x - 32) * 5/9,
    }
    key = (from_unit.lower(), to_unit.lower())
    if key in conversions:
        result = conversions[key](value)
        return {"result": round(result, 4), "from": f"{value} {from_unit}",
                "to": f"{result:.4f} {to_unit}"}
    return {"error": f"Conversion {from_unit} → {to_unit} not supported"}

def word_count(text: str) -> dict:
    """Count words, characters, sentences in text.
    Use when user wants to know length/stats about their writing.
    """
    words = text.split()
    sentences = text.count('.') + text.count('!') + text.count('?')
    return {"words": len(words), "characters": len(text),
            "sentences": sentences, "reading_time_min": round(len(words)/200, 1)}

LlmAgent

If you have 3 tasks that each take 5 seconds and don't depend on each other:

ORDERS = {}

def search_products(query: str, max_price: float = 99999) -> dict:
    """Search for products by name or category. Filter by price.
    Args: query: search terms, max_price: maximum price in rupees
    """
    results = []
    for pid, p in PRODUCTS.items():
        if query.lower() in p["name"].lower() and p["price"] <= max_price:
            results.append({"id": pid, **p})
    return {"products": results, "count": len(results)}

def check_stock(product_id: str) -> dict:
    """Check if a product is in stock.
    Args: product_id: The product ID (e.g., P001)
    """
    if product_id in PRODUCTS:
        p = PRODUCTS[product_id]
        return {"in_stock": p["stock"] > 0, "quantity": p["stock"], "product": p["name"]}
    return {"error": "Product not found"}

def place_order(product_id: str, quantity: int, customer_name: str) -> dict:
    """Place an order for a product.
    Args: product_id: Product to order, quantity: how many, customer_name: buyer name
    """
    import random
    if product_id not in PRODUCTS:
        return {"error": "Product not found"}
    if PRODUCTS[product_id]["stock"] < quantity:
        return {"error": "Insufficient stock"}
    order_id = f"ORD{random.randint(10000,99999)}"
    total = PRODUCTS[product_id]["price"] * quantity
    PRODUCTS[product_id]["stock"] -= quantity
    ORDERS[order_id] = {"product_id": product_id, "quantity": quantity,
                        "customer": customer_name, "total": total, "status": "confirmed"}
    return {"order_id": order_id, "total_inr": total, "status": "confirmed",
            "estimated_delivery": "3-5 business days"}

def track_order(order_id: str) -> dict:
    """Track the status of an existing order.
    Args: order_id: The order ID to track (format: ORDxxxxx)
    """
    if order_id in ORDERS:
        return ORDERS[order_id]
    return {"error": "Order not found"}

🔌 What is MCP?

MCP (Model Context Protocol) is an open standard that lets AI agents connect to external servers — databases, APIs, file systems — using a standardized interface. Think of it as a universal adapter for AI tools.
USB-C is a universal port — any device, any cable, one standard. MCP is the USB-C for AI tools. Any MCP server works with any MCP-compatible AI framework.

async for event in runner.run_async(...):
    # Check for streaming text chunks
    if event.content and not event.is_final_response():
        for part in event.content.parts:
            if hasattr(part, 'text') and part.text:
                print(part.text, end="", flush=True)  # print as it arrives
    
    # Final complete response
    elif event.is_final_response():

🛡️ Error Handling Patterns

Robust tool with error handlingpython
import requests
from requests.exceptions import Timeout, ConnectionError

def robust_api_call(endpoint: str, params: dict) -> dict:
    """Make a robust API call with proper error handling.
    Args: endpoint: API URL, params: Query parameters as dict
    """
    try:
        response = requests.get(
            endpoint,
            params=params,
            timeout=10,   # 10 second timeout
            headers={"User-Agent": "MyAgent/1.0"}
        )
        response.raise_for_status()  # raises for 4xx/5xx
        return {"success": True, "data": response.json()}
    
    except Timeout:
        return {"error": "Request timed out. Please try again."}
    except ConnectionError:
        return {"error": "Cannot connect to service. Check internet."}
    except requests.exceptions.HTTPError as e:
        return {"error": f"Service error: {e.response.status_code}"}
    except Exception as e:
        return {"error": f"Unexpected error: {str(e)}"}

📋 Production Best Practices

  • Always set max_iterations on LoopAgents to prevent infinite loops
  • Use try/except in all tools — never let a tool crash the agent
  • Store API keys in environment variables, never in code
  • Use DatabaseSessionService for persistent sessions in production
  • Add logging to track tool calls and errors in production
  • Set reasonable timeouts on all external API calls (10s default)
  • Write tests using adk eval before deploying changes
  • Monitor with Cloud Logging after deployment
